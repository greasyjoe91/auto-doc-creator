## 🚀 快速开始

### 方式一：自动安装（推荐）

1. **双击运行** `一键部署环境.bat`
2. 脚本会自动：
   - 检测 Node.js 环境
   - 如未安装，自动下载并安装 Node.js
   - 安装项目依赖
   - 启动服务
3. 浏览器自动打开 http://localhost:3005

### 方式二：手动安装

如果自动安装失败，请按以下步骤操作：

#### 1. 安装 Node.js

- 访问 https://nodejs.org/
- 下载 LTS 版本（推荐 v20.18.1）
- 运行安装程序，确保勾选 "Add to PATH"

#### 2. 安装依赖

打开命令行，执行：
```bash
cd server
npm install
```

#### 3. 启动服务

```bash
cd server
npm start
```

访问 http://localhost:3005

## ⚙️ 配置说明

### 必需配置

编辑 `server/.env` 文件，配置 Gemini API Key：

```env
GEMINI_API_KEY=your_encrypted_api_key
```

### 获取 API Key

1. 访问 https://aistudio.google.com/apikey
2. 创建新的 API Key
3. 使用加密工具加密（可选）：
   ```bash
   cd server
   node encrypt-key.ts your_api_key
   ```

### 可选配置

修改 `server/.env` 中的其他配置：

```env
# 服务器端口（默认 3002）
PORT=3002

# CORS 配置
CORS_ORIGIN=http://localhost:3000

# 加密密钥
ENCRYPTION_KEY=your-random-string
```

## 🛠️ 常见问题

### Q1: 端口被占用

**错误**: `Error: listen EADDRINUSE: address already in use :::3002`

**解决**:
1. 修改 `server/.env` 中的 `PORT` 值
2. 或运行 `stop.bat` 停止已有服务

### Q2: API Key 错误

**错误**: `403 Forbidden` 或 `Invalid API Key`

**解决**:
1. 检查 `server/.env` 中的 `GEMINI_API_KEY` 是否正确
2. 确认 API Key 未过期或被撤销
3. 重新生成 API Key

### Q3: 依赖安装失败

**错误**: `npm install` 失败

**解决**:
1. 检查网络连接
2. 尝试使用国内镜像：
   ```bash
   npm config set registry https://registry.npmmirror.com
   npm install
   ```
3. 清除缓存后重试：
   ```bash
   npm cache clean --force
   npm install
   ```

### Q4: Node.js 版本不兼容

**要求**: Node.js 18.0 或更高版本

**解决**:
1. 卸载旧版本 Node.js
2. 重新运行 `一键部署环境.bat`
3. 或手动下载最新 LTS 版本

## 📝 服务管理

### 启动服务

```bash
# 方式1: 使用部署脚本
双击 一键部署环境.bat

# 方式2: 手动启动
cd server
npm start
```

### 停止服务

```bash
# 方式1: 使用停止脚本
双击 stop.bat

# 方式2: 手动停止
# 查找进程
netstat -ano | findstr :3002
# 结束进程
taskkill /F /PID <进程ID>
```

### 查看日志

```bash
# 查看实时日志
type server.log

# 持续监控日志
powershell Get-Content server.log -Wait
```

## 🔒 安全建议

1. **不要提交 .env 文件到版本控制**
2. **定期更换 API Key**
3. **使用加密的 API Key**
4. **限制服务访问权限**（生产环境）
5. **启用 HTTPS**（生产环境）
